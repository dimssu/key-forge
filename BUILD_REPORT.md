# KeyForge — Build Report

A free, universal LLM API key tester. Built end-to-end across 8 specialized agents (specs in `/agents/`).

## Summary

- **Stack**: Next.js 14 App Router · TypeScript strict · Tailwind · shadcn/ui · Framer Motion · Lucide · Zod · @vercel/og · jose
- **Package manager**: pnpm 9
- **Status**: `pnpm typecheck`, `pnpm lint`, `pnpm test` (13/13), and `pnpm build` (25 static pages) all green.
- **Domain**: keyforge.dimssu.com (DNS records below).
- **Auth strategy**: stateless server proxy at `/api/proxy/[provider]`. Keys arrive in `x-keyforge-key`, are forwarded once, never logged or persisted.

## Trust & security audit

- `grep -rn "console.log" app lib components` — only present inside generated *snippet strings* shown in code samples (not runtime logging).
- `grep -rn "localStorage\|sessionStorage" components lib` — used for theme persistence and the brief paste-handoff (consumed and removed).
- Proxy route audit: no `console.*` of headers / bodies; error responses generic; never echoes input.
- Share JWT: only `keyTail = key.slice(-4)` is included.
- Reminder JWT: payload is `{ email, providerId, expiresAt }` — full key never signed.
- OG image route: only reads `title` and `provider` query params.
- Strict CSP, HSTS preload, X-Frame-Options DENY, X-Content-Type-Options nosniff, Referrer-Policy strict-origin-when-cross-origin, Permissions-Policy locked.
- Per-IP rate limit on `/api/proxy/*` (default 30 req/min, configurable).

## Decisions made (no clarifying asks)

| Decision | Choice | Rationale |
| --- | --- | --- |
| Git identity | `dev` / `dev@dimssu.com` | Per spec fallback. No remote URL was provided either; commits are queued locally. |
| Per-provider URL pattern | `/test/<slug>-api-key` | Keyword-rich; SEO-veto-protected over the cleaner `/<slug>` pattern. |
| Default theme | Dark | Dev-focused aesthetic, matches Vercel/Linear inspiration. |
| Composite credentials | JSON-encoded in the same `x-keyforge-key` header | One auth boundary instead of two; the provider adapter parses the JSON. |
| Reminder service | Resend (optional, hidden when env unset) | Lightweight, no DB; fits the stateless story. |
| Status feed | statuspage.io `/api/v2/status.json` | Most providers expose this; "unknown" is acceptable for the rest. |
| Guides as TS modules | `lib/guides-content.ts` instead of MDX | Removes MDX build dependency and contentlayer churn for a content set this small. |
| Test framework | Vitest + Playwright | Vitest is fast for the registry tests; Playwright matches the spec. |

## DNS records for `keyforge.dimssu.com → Vercel`

Add **one** of the following at your registrar (the CNAME is the recommended option for subdomains):

### Option A — CNAME (recommended)

| Type  | Host (or Name) | Value                  | TTL   |
| ----- | -------------- | ---------------------- | ----- |
| CNAME | `keyforge`     | `cname.vercel-dns.com.` | Auto  |

If `dimssu.com` is on Cloudflare, set the proxy mode to **DNS only (grey cloud)** — Vercel manages its own TLS and HSTS. Orange-clouded records will break the certificate provisioning.

### Option B — A records (if your DNS provider does not support CNAME at the apex)

Not relevant for a subdomain. Use Option A.

After the DNS resolves, in **Vercel → Project → Settings → Domains** add `keyforge.dimssu.com` and the project will issue a Let's Encrypt cert automatically.

## Optional environment variables

Set in **Vercel → Settings → Environment Variables** (or `.env.local` for local dev):

| Name | Required? | Purpose |
| --- | --- | --- |
| `NEXT_PUBLIC_SITE_URL` | no | Defaults to `https://keyforge.dimssu.com` |
| `KEYFORGE_SIGNING_SECRET` | optional | Enables `/api/share` and rotation reminder tokens. Generate with `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` |
| `RESEND_API_KEY` | optional | Enables the rotation-reminder UI |
| `REMINDER_FROM_EMAIL` | optional | Defaults to `no-reply@keyforge.dimssu.com` |
| `KEYFORGE_RATE_LIMIT_PER_MINUTE` | optional | Per-IP cap on `/api/proxy/*`. Default 30. |

The site works without any of these — the optional features hide gracefully.
